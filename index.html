SWIFT-TEC v3

Global TEC Forecast (3-day, Base抽出＋加算モデル)
・BoM TEC（最大9枚）から 3日 / 30min 予報を生成
・まず BaseTEC = 入力TEC − Sflare_base − Sstorm_base で静穏ベースを抽出
・その BaseTEC に ΔStorm(Kp予報) ＋ ΔFlare(Deep Flare Net) を足し算
・NOAAテキスト / JSON から Kp(t) を自動読取（Base用と予報用を分離）
・Leaflet 地図で TEC / GPS誤差を確認、地域フォーカスも可能

1. BoM TEC（入力・最大9枚）

World0000_tec.txt など
・API取得は 0000/0300/0600/0900/1200/1500/1800/2100/2330 の計9枚を取得します。
・もし取得に失敗する場合は、ブラウザのCORS制限の可能性があります（その場合は従来通りコピペ運用）。

2. NOAA Kp テキスト（予報用）

3-Day Geomag Forecast を取得可
・Kp（3日予報）は、NOAAの 3-day geomag forecast (text) を自動取得できます。
・このテキストには NOAA Kp index breakdown が含まれるため、現行パーサで安定します。

2b. Base 作成用 Kp / フレア

入力TECから嵐・フレア成分を引き算
・Base用 Kp：NOAA 実測 Kp の JSON 配列を貼付、またはAPI取得
・Base用 フレア：時間ごとの Px / Pm を JSON で入力
※空の場合：Kp は「予報用」を流用、フレアは 0 とみなす


3. Deep Flare Net 領域（予報用）

9分割フレア確率（3×3）
太陽円盤を 3×3 の 9領域に分割し、各領域ごとの Xクラス / Mクラス発生確率 [%] を入力。
